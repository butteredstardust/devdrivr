# App Updater тАФ Design Spec

**Date:** 2026-04-12
**Branch:** feat/app-updater
**Scope:** cockpit app only

## Overview

Add a self-update check feature to the Cockpit desktop app. The app checks GitHub Releases directly for a `latest.json` manifest, compares versions, and lets the user download the correct installer for their platform.

No third-party update server. No Tauri updater plugin (avoids signing key requirement). Direct GitHub artifact downloads.

---

## Settings (3 new toggles in General tab)

| Key | Type | Default | Label |
|-----|------|---------|-------|
| `checkForUpdatesAutomatically` | boolean | `true` | Check for updates automatically |
| `downloadUpdatesAutomatically` | boolean | `false` | Download update automatically |
| `notifyWhenUpdateAvailable` | boolean | `true` | Notify when update is available |

---

## Update Flow

### Auto-check (on startup)
1. Bootstrap completes тЖТ if `checkForUpdatesAutomatically` тЖТ call `checkForUpdate()`
2. Fetch `latest.json` via `@tauri-apps/plugin-http`
3. Compare `latest.version` vs current `getVersion()` from `@tauri-apps/api/app`
4. If newer:
   - If `notifyWhenUpdateAvailable` тЖТ show `<UpdateNotification>` banner
   - If `downloadUpdatesAutomatically` тЖТ download to user's Downloads folder, show toast with path

### Manual accept (from notification banner)
- "Download" button тЖТ `save()` dialog (user picks destination) тЖТ stream download тЖТ toast on completion
- "Dismiss" button тЖТ hide banner for this session

### Manual check (from Settings panel)
- "Check Now" button тЖТ runs `checkForUpdate()` тЖТ shows result inline

## Platform Detection

Rust command `get_platform_info()` returns compile-time `(os, arch)` via `std::env::consts`. Maps to manifest keys:

| Rust | Manifest key |
|------|-------------|
| `(macos, aarch64)` | `darwin-aarch64` |
| `(macos, x86_64)` | `darwin-x86_64` |
| `(windows, x86_64)` | `windows-x86_64` |
| `(linux, x86_64)` | `linux-x86_64` |

---

## CI Change

New `publish-update-manifest` job in `tauri.yml`, runs after all build jobs succeed:
1. Queries release assets via `gh release view cockpit-vVERSION --json assets`
2. Generates `latest.json` with correct asset URLs
3. Uploads `latest.json` to the release via `gh release upload`

---

## Error Handling

- Network failure during check тЖТ silent fail (no toast, check debounced by 1h)
- Network failure during download тЖТ toast error + leave save dialog open
- Platform not in manifest тЖТ toast "Platform not supported for auto-update"
- Same or older version тЖТ no notification
